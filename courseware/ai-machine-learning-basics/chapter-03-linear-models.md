# 第 3 章 线性模型与基线思想

## 3.1 为什么从基线模型开始

基线模型是一个简单、可解释、可快速复现的起点。它不一定追求最高分，而是帮助团队确认数据读取、划分、训练和评估流程是否正确。没有基线，后续复杂模型的改进就缺少比较对象。

## 3.2 线性回归

线性回归假设预测值可以由特征的加权和表示：

```text
y = w1 * x1 + w2 * x2 + ... + b
```

其中 `w` 是权重，`b` 是偏置。权重的符号和大小能帮助我们理解特征对预测结果的影响。

## 3.3 逻辑回归

逻辑回归用于分类任务。它先计算线性得分，再通过 sigmoid 函数把得分映射到 0 到 1 之间，表示属于正类的概率。阈值通常从 0.5 开始，但在样本不均衡场景中需要结合 Precision、Recall 或业务成本调整。

## 3.4 基线报告格式

一个基线报告至少包含：

- 数据来源和样本量。
- 特征和标签定义。
- 划分方式。
- 模型名称与关键参数。
- 验证集指标。
- 与简单规则的比较。
- 当前最大风险和下一步计划。

## 3.5 本章练习

设计一个“平均值预测”或“多数类预测”的朴素基线，再与线性模型比较。解释线性模型是否真的优于朴素规则。
