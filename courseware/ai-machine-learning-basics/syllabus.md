# 机器学习基础与入门预测项目课程大纲

## 课程定位

本课程面向已经具备 Python 基础、但尚未完整经历机器学习项目流程的本科低年级学生。课程不以堆砌算法名称为目标，而是围绕“如何把一个现实预测问题变成可验证的模型项目”组织知识。学生会学习监督学习的基本概念、数据集构造、训练与验证、评估指标、过拟合排查和项目复盘。

## 学习目标

完成课程后，学生应能做到：

1. 用业务问题、输入特征、预测目标和验收指标描述一个监督学习任务。
2. 识别标签泄露、训练/测试混用、样本偏差和指标选择不当等常见风险。
3. 使用 Python 读取表格数据，完成特征选择、训练集/验证集划分和基线模型训练。
4. 根据回归或分类任务选择合适指标，并解释指标变化背后的原因。
5. 用实验记录表复盘模型改进过程，说明下一次迭代计划。

## 章节安排

| 周次 | 主题 | 核心产出 |
|---|---|---|
| 第 1 次 | 机器学习问题定义与数据证据 | 一份问题定义卡 |
| 第 2 次 | 特征、标签与数据划分 | 一份数据字段审查表 |
| 第 3 次 | 线性模型与基线思想 | 一个可运行的基线模型 |
| 第 4 次 | 损失函数、梯度下降与训练循环 | 一份训练过程解释图 |
| 第 5 次 | 评估指标、泛化与错误分析 | 一份评估报告 |
| 第 6 次 | 入门项目整合与复盘 | 一个小型预测项目仓库 |

## 评价方式

课程评价由过程性证据和项目产出共同构成：

- 课堂练习 20%：每章完成一张概念卡或数据检查卡。
- 随堂测验 20%：覆盖概念理解、错误识别和指标解释。
- 实验报告 25%：记录数据处理、训练、评估和问题修复。
- 综合项目 35%：提交代码、结果截图、README 和复盘说明。

## 教学资源要求

教师提供章节讲义、实验指导、题库、项目任务书和拓展阅读清单。学生每次学习都需要留下可检查证据，例如代码提交、指标表、错因记录或复盘文字。系统推荐资源时，应说明推荐依据来自学习画像、当前进度、测验结果或课程资料。
