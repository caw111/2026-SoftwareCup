# 第 1 章 机器学习问题定义与数据证据

## 1.1 什么是可学习的问题

机器学习适合处理“存在历史样本、样本中包含可观察输入、并且有明确目标变量”的问题。一个合格的学习任务至少要回答四个问题：预测什么，输入来自哪里，预测结果如何使用，怎样判断模型是否真的有帮助。

例如“预测明天会不会下雨”可以形成监督学习任务，因为历史天气记录里有温度、湿度、气压等输入，也有第二天是否下雨的标签。相反，“让系统变聪明”不是可学习的问题，因为目标不明确，也无法设计评价指标。

## 1.2 问题定义卡

问题定义卡用于把模糊想法变成可执行任务：

| 字段 | 说明 | 示例 |
|---|---|---|
| 场景 | 模型服务的真实情境 | 校园自习室用电预测 |
| 输入 | 模型可以提前知道的信息 | 日期、星期、天气、历史用电量 |
| 标签 | 需要预测的目标 | 明日每小时用电量 |
| 输出动作 | 预测结果如何影响决策 | 提前安排空调和照明策略 |
| 验收指标 | 判断模型是否可用的标准 | MAE 小于上一周均值基线 15% |

## 1.3 数据证据与标签泄露

数据证据必须在预测时真实可获得。若某个字段在预测时尚不存在，却被放入训练特征，就会造成标签泄露。标签泄露会让离线评估看起来很好，但真实部署时模型无法使用这些信息。

常见泄露包括：把未来结果作为特征，把人工审核后的结论放入输入，把测试集统计量提前用于训练，以及在划分数据前先做全量归一化。

## 1.4 本章练习

选择一个熟悉场景，写出问题定义卡。检查每个输入字段是否在预测时已经存在，并标出至少一个可能导致泄露的字段。
