# 综合项目 校园建筑用电预测

## 项目背景

学校后勤部门希望根据日期、课程安排、天气和历史用电数据预测教学楼次日用电量，以便提前安排节能策略。学生需要构建一个可复现的监督学习项目，并把模型结果转化为可解释的管理建议。

## 数据字段

| 字段 | 含义 | 类型 | 预测时可获得 |
|---|---|---|---|
| date | 日期 | 日期 | 是 |
| weekday | 星期 | 类别 | 是 |
| temperature | 预测温度 | 数值 | 是 |
| humidity | 预测湿度 | 数值 | 是 |
| course_hours | 当日排课小时数 | 数值 | 是 |
| last_day_kwh | 前一日用电量 | 数值 | 是 |
| next_day_kwh | 次日用电量 | 数值 | 否，标签 |

## 项目任务

1. 写出问题定义卡，明确输入、标签、输出动作和验收指标。
2. 完成数据字段审查，删除预测时不可获得字段。
3. 构建均值基线和线性回归基线。
4. 至少尝试一种特征改进，例如星期独热编码或温度分段。
5. 生成评估报告，包含 MAE、误差最大的 5 个样本和下一步改进计划。
6. 提交 README，说明如何复现实验。

## 评分 Rubric

| 维度 | 优秀 | 合格 | 需改进 |
|---|---|---|---|
| 问题定义 | 预测点、标签、指标和业务动作清晰 | 能描述基本任务 | 目标模糊 |
| 数据处理 | 有字段审查和泄露检查 | 能完成基本读取 | 混用标签或测试集 |
| 模型训练 | 有基线对比和实验记录 | 能训练一个模型 | 无验证集 |
| 评估复盘 | 指标解释结合业务 | 只报告指标 | 只给截图无解释 |
