# LearnMate 成熟功能扩展设计

## 设计边界

- 沿用现有 LearnMate 的蓝灰色视觉、三栏课程布局、卡片、状态标签、时间线和 Markdown 内容体系。
- 课程侧栏只承担一级导航；相关功能通过分组折叠，避免把十四个入口平铺成冗长菜单。
- 新功能优先复用现有 `plan.data`、任务进度、诊断、测验和 Agent 产物，不维护相互冲突的第二份业务状态。
- 所有模型结果必须有本地确定性回退；所有写入接口必须保留匿名用户隔离。
- 每项功能先增加数据契约和测试，再接入界面；每项完成后运行专项测试与完整 `npm test`。

## 1. 课程能力导航

### 信息架构

- 核心学习：学习路径、我的笔记。
- 学情分析：课前测、掌握度、个性化补救。
- 实践评测：随堂测验、错题复习、综合考试、项目任务。
- 成果与系统：学习报告、质量治理、Agent 协作、学习设置。
- “获得帮助”继续保留为右侧上下文入口，避免和课程功能混排。

### 交互约束

- 分组默认展开当前页面所属组，用户可折叠非当前组。
- 当前页面通过现有 `.active` 状态和 `aria-current` 标识。
- 小屏使用横向可滚动的紧凑入口，不复制第二套业务导航。

## 2. 对话式画像构建

### 数据模型

- `profileInterview.messages`：学生与画像智能体对话。
- `profileInterview.draft`：主题、目标、水平、专业、周期、每日时长、偏好、薄弱点、学习历史。
- `profileInterview.completeness`：字段完整度、缺失项和下一问。
- `profileInterview.profilePreview`：六维画像、置信度、证据来源和策略。

### 流程

1. 学生先用自然语言描述需求。
2. 画像智能体抽取已知字段，只追问真正缺失或低置信字段。
3. 右侧实时展示六维画像、证据和完整度。
4. 学生确认后才进入课程生成；仍可打开“高级设置”精调结构化字段。

### LLM 交互设计

- 浏览器只提交最近对话、当前画像草稿和本轮输入；API 密钥始终保留在服务端，由统一 LLM 客户端发起请求。
- 模型必须同时返回自然语言回复和结构化画像 JSON。服务端对白名单字段、枚举值、长度、置信度和证据片段进行二次校验，再更新画像，不能让模型响应直接进入页面状态。
- 每轮先复述本轮识别到的关键信息，再最多追问一个最有价值的问题；画像足够完整时主动邀请用户确认，不重复机械询问已知信息。
- 对话保留最近上下文，支持用户用“改成”“不是”“重新调整”等表达修正旧画像；服务端仍用本地抽取结果补强时间、枚举等确定字段。
- 页面明确展示“LLM 对话 / 本地降级”状态。模型未配置、超时、无效 JSON 或上游失败时保留本轮输入并使用确定性规则继续交互，同时显示可恢复提示，不能伪装成 LLM 回复。

## 3. 课程知识库与 RAG 引用

### 数据模型

- `course_sources`：文件元数据、解析状态、字符数、校验和。
- `course_source_chunks`：章节、页码/段落、正文、关键词和后续扩展检索向量的轻量 JSON 表示。
- `plan_sources`：课程与来源绑定。
- 生成内容中的引用统一为 `citations: [{sourceId, chunkId, title, locator, quote}]`。

### 支持范围

- 首批支持 TXT、Markdown、CSV、JSON 和可抽取文本的 PDF；文件大小、类型和内容长度均在服务端校验。
- 资料问答、课程生成、每日资料和导师回答默认读取所选文件的全部已解析内容并一次性提交给 LLM；超过单次模型上下文上限时明确报错，要求减少文件，不静默截断。
- 旧的分块检索接口保留为诊断与后续向量数据库扩展入口，但不作为当前 LLM RAG 问答链路。

### LLM RAG 生成设计

- 完整问答链路为“用户问题 → 用户/课程范围校验 → 读取所选文件完整解析内容 → LLM 全文资料回答 → 引用白名单校验 → 只返回实际使用的引用”，不能把文件卡片或本地摘录称为模型回答。
- 模型以结构化 JSON 返回 Markdown 回答、实际使用的引用编号、证据覆盖度和后续问题；回答中的 `[S1]` 等编号必须存在于本轮所选文件全文上下文，未知编号、无引用事实回答或无效 JSON 均进入一次自动修复。
- 资料正文按不可信数据处理，忽略其中改变角色、索取密钥、覆盖系统规则等提示注入内容。资料不足时必须明确说明，不允许用模型常识伪装成资料结论。
- 上游瞬时故障自动重试；模型不可用时显示“抽取式降级”，仅摘录真实文件内容并保留引用，绝不伪装为 LLM RAG。
- 课程生成、每日讲义和导师问答都记录 `llmUsed`、读取文件数、读取内容块数、全文字符数、候选引用和模型实际使用的引用，便于展示和审计。
- 页面展示服务端 `traceId`、LLM 调用次数和“文件全文、答案生成、引用校验”三阶段状态；浏览器只请求业务 API，绝不直接持有或调用模型密钥。

## 4. 动态路径重规划

### 触发信号

- 诊断完成、连续错题、综合考试、补救复测和任务逾期。

### 输出

- 不直接覆盖原路径，生成 `pathRevisions`，包含触发证据、修改前后差异、插入/调整/延期动作和可撤销版本。
- 重规划会更新未完成学习日，绝不修改已完成任务或学习笔记。
- 前端以“路径变更摘要 + 前后对比 + 接受/撤销”呈现。

## 5. 多智能体真实返工闭环

### 执行模型

- 每个 Agent 产出版本化 artifact，并记录输入、输出、校验、耗时和状态。
- 治理智能体返回结构化 finding；严重 finding 会创建 revision request。
- 资源智能体按 finding 重写目标 artifact，最多两轮；每轮重新校验。
- 流式事件包含 `artifact-created`、`review-failed`、`revision-start`、`revision-done`。

## 6. 交互式知识图谱

- 使用原生 SVG，避免引入与现有系统风格冲突的大型图形库。
- 支持拖拽、缩放、按维度/掌握度筛选、节点搜索、先修路径高亮和详情侧板。
- 节点详情展示目标、误区、掌握证据、关联资源、复习时间和下一步动作。
- 列表视图作为无障碍与小屏回退。

## 7. 一键思维导图

- 从知识图谱和学习路径生成可编辑的层级树，而不是静态截图。
- 支持展开/折叠、聚焦分支、主题色、SVG 下载、打印和复制大纲。
- 生成结果保存到课程产物中，可由 Agent 治理检查节点覆盖率。

## 8. 学习日历、热力图、徽章与真实连续天数

### 事件模型

- 所有任务、测验、诊断、导师、项目和报告事件统一形成 `learning_activity_events`。
- 连续学习按用户时区的自然日计算，同日多次活动只计一天。

### 界面

- 课程概览增加真实连续天数和今日活动强度。
- 学习设置页增加月历、近一年热力图、活动明细和复习到期事项。
- 徽章由可解释规则驱动，包含解锁时间、进度和下一目标，不只显示静态图标。

## 测试策略

- 后端：纯函数单元测试、Repository 契约测试、API 集成测试。
- 前端：DOM/源码契约测试覆盖导航、对话、图谱、思维导图和日历结构。
- 回归：每个功能后运行全量 `npm test`；最后验证健康检查、数据库、判题和关键 API。
